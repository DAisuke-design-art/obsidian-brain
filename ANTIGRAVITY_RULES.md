# Antigravity & Agent Operational Rules (STRICT)

このファイルは、本ワークスペースにおけるAIエージェント（Antigravity）の行動指針を定めた「絶対ルール」である。
AIは作業開始前に必ずこのファイルの内容を確認し、これを遵守しなければならない。

## 1. プロジェクト基本構造（Single Source of Truth）
- **ルートディレクトリ（絶対聖域）**: `/Users/daisukemiyauchi/Documents/Obsidian`
- **概要**: ここはユーザーの思考・知識・日記を管理するObsidian保管庫（Vault）であり、同時にGitリポジトリである。
- **禁止事項**: 
    - `~/Documents/GitHub/obsidian-brain` などの別ディレクトリを作成・参照してはならない。
    - 複製（クローン）を作成してはならない。常に現在のルートディレクトリ内のファイルを直接編集すること。

## 2. Git運用ルール（Manual Control）
- **運用方針**: Gitによるバージョン管理は**「ユーザーの手動操作」**を原則とする。
- **AIの権限**:
    - ユーザーから明示的な指示がない限り、`git add`, `git commit`, `git push` を自動実行してはならない。
    - コンフリクト発生時は、勝手に解決せず、必ずユーザーに状況を報告し指示を仰ぐこと。
- **同期**: リモートリポジトリ（origin）は正しく設定済みである。

## 3. 会話ログの管理（Logging）
- **ログファイル**: `Gemini_Log.md`
- **運用**: 
    - ユーザーは `glog` コマンド（`Scripts/glog.sh`）を使用して、ターミナル経由でチャットを行う場合がある。
    - このスクリプトは「会話ログの記録」のみを行い、Gitの自動同期機能は削除済み（手動運用）である。
    - AIはこのログファイルを参照することで、過去のコンテキストを把握できる。

## 4. 思考と振る舞い
- **シビアな視点**: 曖昧な推測で行動せず、不明点は必ずユーザーに確認すること。
- **安全性**: 重要なファイル（過去の日記や設定ファイル）を削除・上書きする際は、必ずバックアップの確認を行うこと。
